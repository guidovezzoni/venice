# 5.1.1 — Open Google Maps Navigation to a Stop

**Epic:** Navigation Handoff
**Feature:** Launch Navigation

## User Story

As a user, I want to tap a stop and open Google Maps navigation to it so that I get turn-by-turn directions without the app having to build its own navigation.

## Description

Each stop in the trip overview has a "Navigate" action. Tapping it opens Google Maps (or the default navigation app) with the stop's coordinates as the destination. The app delegates all turn-by-turn navigation to Google Maps.

## Acceptance Criteria

- Each stop (except the starting point if already departed) has a "Navigate" action.
- Tapping it opens Google Maps in navigation mode with the stop as the destination.
- If Google Maps is not installed, the app falls back to a `geo:` URI intent that other map apps can handle.
- The action uses the stop's stored latitude and longitude, not a text search.

## Technical Notes

- **Implementation:** Fire an Android Intent with a `google.navigation` URI:
  ```
  google.navigation:q={lat},{lng}
  ```
  This opens Google Maps directly in navigation mode.
- **Fallback:** Use a `geo:{lat},{lng}?q={lat},{lng}(Label)` URI as a generic fallback for devices without Google Maps.
- This is a single-line intent launch — no SDK integration or API key needed.
