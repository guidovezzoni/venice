# 7.1.1 — See Roadtrip on Android Auto

**Epic:** Android Auto *(future)*
**Feature:** Auto Trip View

## User Story

As a user, I want to see my current roadtrip and upcoming stops on Android Auto so that I can glance at my trip progress while driving.

## Description

The app provides a read-only view of the active roadtrip on the Android Auto car display. It shows a list of upcoming stops with distance/duration for each leg. The view uses Android Auto's templated UI — no custom map rendering. The user plans the trip on the phone; Auto simply mirrors the current state.

## Acceptance Criteria

- The app appears in the Android Auto app launcher under the POI category.
- When opened, it shows the active trip's upcoming stops (max 6 visible per Auto constraints).
- Each stop shows its name, distance, and duration from the previous stop.
- Departed stops are either hidden or visually deprioritised.
- The view updates if trip data changes on the phone (e.g., stop marked as departed).

## Technical Notes

- **Library:** Android for Cars App Library (`androidx.car.app`), using `ListTemplate` or `PlaceListMapTemplate`.
- **Category:** POI (Point of Interest) — the app shows a list of places the user can navigate to.
- **Constraints:** Auto templates limit visible list items to ~6. For trips with more stops, show the current stop and the next 5 upcoming stops.
- **Architecture:** The Auto UI is a completely separate UI layer from the phone's Compose UI. It consumes the same `TripRepository` and domain models. This is why clean separation of UI and domain logic (established in Level 1) is critical.
- **No editing:** Users cannot add, remove, or reorder stops from the Auto interface. All trip management happens on the phone.
