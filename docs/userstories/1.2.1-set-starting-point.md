# 1.2.1 — Set a Starting Point

**Epic:** Trip Management
**Feature:** Stop Management

## User Story

As a user, I want to set a starting point for my roadtrip so that the route calculation knows where I'm departing from.

## Description

The user designates a location as the first stop of the trip. The starting point is selected via place search (see Epic 2) and stored with its name and coordinates. A trip must have a starting point before a route can be calculated.

## Acceptance Criteria

- The user can set a starting point via place search.
- The starting point is stored as the first stop (order = 0) with name, latitude, and longitude.
- The starting point is visually distinguished from intermediate stops in the UI.
- The starting point can be changed after initial selection.

## Technical Notes

- **Data model:** `Stop(id, tripId, placeName, latitude, longitude, order, status)` — the starting point is simply the stop with `order = 0`.
- Depends on place search (2.1.1) for location selection.
- Store `latitude` and `longitude` on every stop from the start — this is critical for Level 2 (GPS-based ETA) where the Directions API is called with live coordinates as origin.
