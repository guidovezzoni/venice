# 6.1.2 — Recalculate ETAs Based on Current Position

**Epic:** Live Position *(future — Level 2)*
**Feature:** GPS-based ETA

## User Story

As a user, I want ETAs to recalculate based on my current position so that I see accurate arrival times even if I deviate from the planned route.

## Description

Building on 6.1.1, the Directions API response from the live GPS position includes updated duration estimates that account for the user's actual location — including any detours, wrong turns, or route changes. The app recalculates ETAs for the current leg and, optionally, for remaining legs.

## Acceptance Criteria

- ETA to the next stop reflects the user's actual position, not the last departed stop.
- If the user deviates from the planned route, the ETA adjusts accordingly.
- ETAs for subsequent legs (beyond the next stop) remain static (pre-calculated) unless the user requests a full recalculation.
- Total remaining time is displayed as: live ETA to next stop + static durations for remaining legs.

## Technical Notes

- Only the current leg needs live recalculation. Future legs are still served from stored `Leg.durationSeconds` values. This keeps API costs proportional to driving time, not trip length.
- The Directions API response will include a new route if the user has deviated. The app does not need to render this route in Level 2, but the duration/distance values should be used.
- Consider caching the last live response to avoid redundant calls when GPS jitters without meaningful displacement.
