# 4.1.1 — Trip Overview with Per-Leg Breakdowns

**Epic:** Trip Overview
**Feature:** Trip Summary View

## User Story

As a user, I want to see a trip overview with all stops and per-leg breakdowns so that I have a complete picture of my roadtrip at a glance.

## Description

The trip overview is the main detail screen for a trip. It shows an ordered list of all stops with the distance and duration of each leg between them, plus the trip totals. It also reflects stop progress (departed/current/upcoming). This is the central screen of the app — the hub from which the user edits stops, triggers route calculation, and launches navigation.

## Acceptance Criteria

- All stops are listed in order, from starting point to destination.
- Between each pair of consecutive stops, the leg's distance and duration are shown.
- Total distance and total duration are displayed (see 3.2.1, 3.2.2).
- Stop progress status is visually indicated (see 1.3.2).
- If the route has not been calculated or is stale, a clear prompt to calculate/recalculate is shown.
- The screen provides entry points to: add/edit/remove/reorder stops, calculate route, navigate to a stop.

## Technical Notes

- This screen aggregates data from multiple domain objects: `Trip`, `Stop[]`, `Leg[]`.
- **UI structure suggestion:**
  ```
  ┌──────────────────────────────┐
  │ Trip Name            [Edit]  │
  ├──────────────────────────────┤
  │ ✓ Rome (departed)            │
  │   │  2h 15m · 195 km         │
  │ → Florence (current)         │
  │   │  1h 05m · 105 km         │
  │ ○ Bologna                    │
  │   │  2h 10m · 210 km         │
  │ ○ Milan                      │
  ├──────────────────────────────┤
  │ Total: 510 km · 5h 30m       │
  ├──────────────────────────────┤
  │ [+ Add Stop]  [Calculate]    │
  └──────────────────────────────┘
  ```
- This is the screen that Android Auto will later present a simplified, read-only version of (see 7.1.1). Keep the data layer clean so Auto can consume the same Trip/Stop/Leg models.
