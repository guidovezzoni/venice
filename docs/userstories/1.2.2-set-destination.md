# 1.2.2 — Set a Destination

**Epic:** Trip Management
**Feature:** Stop Management

## User Story

As a user, I want to set a destination for my roadtrip so that the route calculation knows where I'm heading.

## Description

The user designates a location as the final stop of the trip. The destination is selected via place search and stored with its name and coordinates. A trip must have both a starting point and a destination before a route can be calculated.

## Acceptance Criteria

- The user can set a destination via place search.
- The destination is stored as the last stop (highest order) with name, latitude, and longitude.
- The destination is visually distinguished from intermediate stops in the UI.
- The destination can be changed after initial selection.
- Adding intermediate stops does not displace the destination — it always remains last.

## Technical Notes

- Uses the same `Stop` data model as 1.2.1. The destination is the stop with the highest `order` value.
- When intermediate stops are added (1.2.3) or reordered (1.2.4), the destination's order must be recalculated to remain last.
