# 7.1.2 — Navigate to a Stop from Android Auto

**Epic:** Android Auto *(future)*
**Feature:** Auto Trip View

## User Story

As a user, I want to tap a stop on Android Auto to start navigation so that I can get directions without touching my phone.

## Description

Each stop in the Android Auto trip view has a "Navigate" action. Tapping it launches Google Maps navigation on the car display with the stop's coordinates as the destination. This is the same handoff pattern as the phone (5.1.1) but triggered from the Auto UI.

## Acceptance Criteria

- Each upcoming stop in the Auto trip view has a navigate action.
- Tapping it launches turn-by-turn navigation to the stop's coordinates.
- Navigation opens in the car display's navigation app (typically Google Maps).
- The action works without the user needing to interact with the phone.

## Technical Notes

- **Implementation:** Use the Car App Library's `NavigationManager` or fire a navigation intent via `CarContext`. The exact mechanism depends on the Auto API level.
- **POI apps and navigation:** POI-category apps can trigger navigation to a place. The platform handles the handoff to the navigation app (Google Maps).
- Since the phone app already stores stops with lat/lng coordinates, the Auto layer just reads the same data and fires the navigation intent — minimal additional logic needed.
