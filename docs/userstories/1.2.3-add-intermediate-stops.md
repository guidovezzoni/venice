# 1.2.3 — Add Intermediate Stops

**Epic:** Trip Management
**Feature:** Stop Management

## User Story

As a user, I want to add intermediate stops to my roadtrip so that I can plan breaks, visits, and detours along the way.

## Description

The user can add stops between the starting point and destination. Each stop is selected via place search and inserted at a specific position in the stop order. Roadtrips are fundamentally about the stops along the way, so this is a core interaction.

## Acceptance Criteria

- The user can add a new stop between any two existing stops.
- The stop is selected via place search and stored with name, latitude, and longitude.
- The order of existing stops adjusts to accommodate the new stop.
- The destination always remains the last stop.
- The user can add up to 23 intermediate stops (25 total including start and destination — Google Directions API waypoint limit).

## Technical Notes

- Google Directions API supports up to 25 waypoints per request. This includes origin and destination, leaving room for 23 intermediate stops. Enforce this limit in the UI.
- After adding a stop, any previously calculated legs are invalidated and should be recalculated (see 3.1.1).
