# 2.1.2 — Resolve Place Names to Coordinates

**Epic:** Place Search
**Feature:** Geocoding

## User Story

As a user, I want the app to resolve place names to coordinates so that routes can be calculated between my stops.

## Description

When a place is selected from search results, the app obtains its geographic coordinates (latitude, longitude). These coordinates are stored with the stop and used as input to the Google Directions API for route calculation.

## Acceptance Criteria

- Every stop has a valid latitude and longitude after place selection.
- Coordinates are obtained automatically as part of the place selection flow — no manual input from the user.
- If coordinate resolution fails (network error, ambiguous result), the user is informed and can retry.

## Technical Notes

- **API:** When using Places Autocomplete (2.1.1), the Place Details endpoint returns coordinates for the selected place. This is a single follow-up call after the user picks a suggestion.
- **Flow:** User types → Autocomplete returns predictions → User picks one → Place Details returns lat/lng → stored on Stop.
- Coordinates must be stored as `Double` (not `Float`) to preserve precision for the Directions API.
