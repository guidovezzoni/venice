# 2.1.1 — Search for a Place by Name

**Epic:** Place Search
**Feature:** Geocoding

## User Story

As a user, I want to search for a place by name when adding a stop so that I don't have to know exact coordinates.

## Description

When adding or editing a stop, the user types a place name (city, address, landmark) and sees a list of matching results. They select one, and its name and coordinates are used for the stop. This is the primary way users will input locations.

## Acceptance Criteria

- A search field is presented when adding or editing a stop.
- As the user types, matching places are suggested (autocomplete).
- Each suggestion shows the place name and a disambiguating detail (e.g., region, country).
- The user selects a suggestion to confirm the place.
- The selected place's name, latitude, and longitude are stored on the stop.

## Technical Notes

- **API:** Google Places Autocomplete (part of the Google Maps Platform). Returns place predictions as the user types, with minimal API calls via debouncing.
- **Pricing:** Places Autocomplete is billed per session (not per keystroke) when using session tokens — roughly $0.017 per session. Well within the $200/month free credit during development.
- **Alternative:** Google Geocoding API is simpler (single request, no autocomplete) but worse UX. Autocomplete is strongly recommended.
- This is a cross-cutting dependency — stories 1.2.1, 1.2.2, 1.2.3, and 1.2.5 all rely on this.
