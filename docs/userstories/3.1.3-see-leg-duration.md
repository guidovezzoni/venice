# 3.1.3 — See Estimated Duration for Each Leg

**Epic:** Route Calculation
**Feature:** Leg Calculation

## User Story

As a user, I want to see the estimated duration for each leg of the trip so that I can plan my time.

## Description

After route calculation, each leg displays the estimated driving time between its two stops. Duration is shown in a human-readable format (e.g., "2h 15m").

## Acceptance Criteria

- Each leg in the trip view shows its estimated duration.
- Duration is formatted as hours and minutes (e.g., "2h 15m" or "45m" for sub-hour legs).
- If the route has not been calculated yet, duration is shown as unavailable (e.g., "—").
- Duration reflects typical driving conditions at the time of calculation (the API factors in average traffic by default).

## Technical Notes

- The Google Directions API returns duration in seconds (`legs[].duration.value`). Store as integer seconds in the database; format for display in the UI layer.
- The API also returns `duration_in_traffic` when a departure time is specified. For Level 1, the default `duration` (without traffic) is sufficient. Traffic-aware durations could be a future enhancement.
