# 1.3.1 — Mark a Stop as Visited/Departed

**Epic:** Trip Management
**Feature:** Stop Progress

## User Story

As a user, I want to mark a stop as visited/departed so that I can track my progress along the roadtrip.

## Description

While on the trip, the user can manually mark a stop as completed/departed. This updates the stop's status and shifts the "current position" in the trip to that stop. In Level 1, this is a manual action; in Level 2, it could be augmented with GPS proximity detection.

## Acceptance Criteria

- Each stop has a status: `upcoming`, `current`, or `departed`.
- The user can mark the current stop as departed.
- Marking a stop as departed automatically sets the next stop as `current`.
- The starting point can be marked as departed (the trip has begun).
- The action is reversible — the user can undo marking a stop as departed.

## Technical Notes

- **Data model:** Add `status` enum to `Stop`: `UPCOMING | CURRENT | DEPARTED`.
- The concept of "current position = last departed stop" is the foundation for Level 2 GPS upgrades. In Level 1, `currentPosition = lastDepartedStop.latLng`. In Level 2, `currentPosition = gps.latLng`. Abstract this behind a position provider interface to make the transition clean.
